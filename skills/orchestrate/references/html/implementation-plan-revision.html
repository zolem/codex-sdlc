What changed in v2

Normalize on blur, not on every keystroke.

In v1 the store was rewritten to E.164 the moment input parsed successfully — on every keystroke. The engineer flagged that as a UX risk (cursor jumps, paste behavior, controlled inputs flickering). The architect re-evaluated and flipped the call: validation still runs on change so errors surface live, but the store only takes the normalized value on blur and on submit.

I don't want the store rewritten while the user is mid-typing. Validate live for the error message, but only normalize the stored value on blur. The open question in §06 already flagged this — let's resolve it that way. — Engineer review of v1 · 2026-05-16

04

The schema, in full

Everything else in this plan is wiring. The Zod schema is the actual feature — if this looks right, the rest of the plan likely does too. 

// apps/web/src/utils/phone/us-sms-phone.schema.ts
import { z } from "zod";

// Output shape when accepted: E.164 US — "+1" followed by exactly 10 digits.
export type UsSmsPhone = `+1${string}`;

const stripToDigits = (s: string) => s.replace(/\D/g, "");

export const optionalUsSmsPhoneSchema = z
  .string()
  .optional()
  .transform((raw) => (raw ?? "").trim())
  .superRefine((trimmed, ctx) => {
    if (trimmed === "") return;
    const digits = stripToDigits(trimmed);
    if (digits.length === 0)
      ctx.addIssue({ code: "custom", message: "Please enter a phone number." });
    else if (digits.length < 10)
      ctx.addIssue({ code: "custom", message: "That number looks too short." });
    else if (digits.length === 11 && !digits.startsWith("1"))
      ctx.addIssue({ code: "custom", message: "We can't recognize that number." });
    else if (digits.length > 11)
      ctx.addIssue({ code: "custom", message: "That number has too many digits." });
  })
  .transform((trimmed): UsSmsPhone | undefined => {
    if (trimmed === "") return undefined;
    const digits = stripToDigits(trimmed);
    return (digits.length === 11 ? `+${digits}` : `+1${digits}`) as UsSmsPhone;
  });

export const isSmsPhoneStepComplete = (
  phone: string | undefined,
  consent: boolean,
): boolean => {
  if (!consent) return false;
  const r = optionalUsSmsPhoneSchema.safeParse(phone);
  return r.success && typeof r.data === "string";
};
