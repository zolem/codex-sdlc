us-sms-phone.schema.ts:1–7

The contract

Sets the shape callers will see. UsSmsPhone is a template-literal type — only +1XXXXXXXXXX strings inhabit it, so accidental misuse becomes a type error rather than a runtime bug. stripToDigits is named so the regex isn't repeated and the intent ("keep digits only") is visible.

+import { z } from "zod";
+
+// Output shape when accepted: E.164 US — "+1" followed by 10 digits.
+export type UsSmsPhone = `+1${string}`;
+
+const stripToDigits = (s: string) => s.replace(/\D/g, "");
us-sms-phone.schema.ts:9–32

Preprocess → refine → transform

A three-phase pipeline. The first transform trims the input so the optional-empty path is uniform. superRefine emits four distinct user-facing messages — the form can tell the user what is wrong, not just that something is. The final transform returns undefined for the legitimately-empty case and a normalized E.164 string otherwise. The 11-digit branch handles the common "1XXXXXXXXXX" paste case without doubling the country code.

+export const optionalUsSmsPhoneSchema = z
+  .string()
+  .optional()
+  .transform((raw) => (raw ?? "").trim())
+  .superRefine((trimmed, ctx) => {
+    const digits = stripToDigits(trimmed);
+    if (digits.length === 0)
+      ctx.addIssue({ code: "custom", message: "Please enter a phone number." });
+    else if (digits.length < 10)
+      ctx.addIssue({ code: "custom", message: "That number looks too short." });
+    else if (digits.length === 11 && !digits.startsWith("1"))
+      ctx.addIssue({ code: "custom", message: "We can't recognize that number." });
+    else if (digits.length > 11)
+      ctx.addIssue({ code: "custom", message: "That number has too many digits." });
+  })
+  .transform((trimmed): UsSmsPhone | undefined => {
+    if (trimmed === "") return undefined;
+    const digits = stripToDigits(trimmed);
+    return (digits.length === 11 ? `+${digits}` : `+1${digits}`) as UsSmsPhone;
+  });

Note: the empty-string fast-path inside superRefine is missing here — that's the bug Stop 4 fixes.

us-sms-phone.schema.ts:34–41

Selector for the Continue gate

Stop 3 wires this into the onboarding flow's disabled prop. Combines consent and a successful parse — both required. Returns a boolean only; the caller never sees the parse result, which keeps the type narrow.

+export const isSmsPhoneStepComplete = (
+  phone: string | undefined,
+  consent: boolean,
+): boolean => {
+  if (!consent) return false;
+  const r = optionalUsSmsPhoneSchema.safeParse(phone);
+  return r.success && typeof r.data === "string";
+};
