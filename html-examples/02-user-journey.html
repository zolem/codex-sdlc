SDLC artifact · Level Up 3.3

Teach a new teacher to write a custom prompt — and stop when their Space exists

Level Up step 3.3 is currently a placeholder. This change ships a focused four-stop tour: point at the sidebar, walk through the Space-creation page, emphasize the prompt field, and mark the step complete the moment the Space is created. No tour of "preview as student" or "ready to launch" — that's BUILD_SPACE's job.

GUIDE.CUSTOM_PROMPT Reuses existing beacon system No DB / API changes Launch-flag activation only

The four stops the user will see — click any to expand

1

Highlight the sidebar entry

on: /level-up
A beacon points at "Create a Space" in the sidebar. Clicking it advances.
data-guide-anchor="nav-create-space"
Reuses the same anchor as BUILD_SPACE step 1. Returns null if the sidebar isn't on screen — no orphan beacons. Tracking id: custom-prompt-guide-open-sidebar.
2

Land on the create page

on: /dot/spaces/create
Beacon emphasizes the prompt input — copy explicitly says "custom prompt".
data-guide-anchor="space-create-prompt"
Reuses the existing empty-prompt guard so the user can't Next-skip past a blank prompt. Step shown while prev ≤ 2; capped at 3 once they advance, so the panel doesn't jump during the async create + navigation.
3

Click submit

on: /dot/spaces/create
Beacon points at the submit button. Clicking it kicks off Space creation.
data-guide-anchor="space-create-submit"
Mirrors BUILD_SPACE step 3. If create fails, the user stays on the page and the beacon stays put — they can retry.
✓

Mark complete — silently

after: router.push to /dot/spaces/create/[spaceId]
Once navigation to the new Space resolves, the guide calls complete() and disappears.
trigger: handleCreate success + router.push resolved
No extra "Next" click required. The existing celebration UI from LevelUpGuideProvider handles the visual reward. On the wizard route (/.../create/[spaceId]) this spec returns null for every step — so even if completion lags by a frame, no beacon appears there.
Completion boundary: Tour ends the moment the Space is created. "Save and continue", "preview as student", and "ready to launch" are not part of this guide — they belong to BUILD_SPACE.

How this guide differs from BUILD_SPACE

Custom Prompt (this work)

  • 3 instructional steps, completes on create
  • Copy spotlights the prompt
  • Returns null on wizard routes
  • Dedicated i18n namespace customPromptGuide.*
  • Tracking prefix custom-prompt-guide-*

Existing build-space guide

  • Continues through wizard + launch
  • Generic "build a Space" framing
  • Resolves steps 4+ on wizard routes
  • buildSpaceGuide.* strings
  • build-space-guide-* tracking

Risks we're tracking

High
Creation level may stay hidden in production.
Step 3.2 is still a placeholder. Until it gets a real launch, the Creation level won't appear in VISIBLE_LEVELS — so users won't see this guide even after ship.
Med
First step lands on a route without the sidebar.
resolveStep returns null — no orphan beacon, but the user also won't know what to do. May need a fallback launch href if analytics show drop-off.
Low
Both guides eligible on the same URL.
Single session flag in practice. Registry order has custom-prompt winning on the shared landing route as a belt-and-suspenders measure.
Low
Double celebrate.
Guarded by checking isGuideActive before calling complete().
