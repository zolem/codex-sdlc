Level Up: a 15-step gamified onboarding for teachers

Teachers — new sign-ups and existing users alike — get a "Level Up" tab in the sidebar, a profile-header certification badge, and an accordion of 5 sequential levels with 3 steps each. Steps launch existing beacon walkthroughs; completions persist per-step. Ships behind a feature flag, off by default.

Scope

In

  • Sidebar nav item + /level-up route, flag-gated
  • Profile header with certification badge
  • 5-level accordion, sequential unlock
  • Reuse Bellringer + Spaces intro for L1S2, L1S3
  • 15 new guide names registered
  • L5S3 uses a manual "mark as done"
  • Localized strings via /en/translation.ts

Not in this release

  • Certificate generation / download
  • Auto-detection of prior feature usage
  • Celebration animations on level complete
  • Analytics events
  • Non-teacher roles (students, admins)
  • Skipping or reordering levels

Risks we're tracking

High
L5S3 "Attend a live training" has no clean completion trigger.
Plan: manual "mark as done" button. Webhook-driven detection is a possible follow-up.
Med
Reused guides also record their own completions under different names.
Decision needed: wire a Level Up–scoped completion in parallel, or share names?
Med
Existing users with deep feature usage start at Level 1.
Intentional for v1 simplicity. May feel patronizing — qualitative feedback will tell.
Low
tRPC shape compatibility with older extension bundles.
Constraint: additive-only changes to guideCompletions.*.

How we'll know it works

Test plan covers the 24 user stories across unit (state derivation, level/step logic), integration (tRPC wiring, flag gating, completion mutations), and end-to-end (full guided walkthrough per step, persistence across sessions). Non-functional checks include keyboard navigation, ARIA state announcements on locked/active/complete, and bundle-boundary compatibility for older extensions.
